Introduction

Why study/use Matlab?

  • a bit of history
    • Created in the later 1970 by Cleve Moler on the basis of the EISPACK.
    • In 1985 became a commercial software package.
    • Now it is de facto a standard for research in numerical linear algebra.
  • main features
    • interpreted high-level language, specialized for linear algebra
    • efficient low-level computations thanks to the use of the BLAS and LAPACK (successor of EISPACK)
    • a wide range of functions, organized in toolboxes (some of them are also commercial products)
  • disadvantages and alternatives
    • not free (free alternative: GNU Octave)
    • less efficient in comparison with low-level languages (more efficient alternative: Julia)
    • not a general purpose programming language (popular alternative: Python)

Prepare your work environment

  • install Matlab/Octave on your PC
    • in order to use Matlab, you need a license; VUB offers a group license for students
    • for our purposes (linear algebra, signals, and systems), Octave is sufficient
    • Octave does not have an equivalent to Simulink (graphical simulation package). Also there are incompatibilities between Matlab and Octave when using toolboxes.
  • use your favorite text editor or IDE
    • You don't have to use the build-in editor in Matlab.
    • GNU emacs is a powerful text editor. It has a steep learning curve.

    • Other popular options: vim, atom, notepad++, gedit

      The good craftsman are recognized by their tools.

  • Matlab is an interpreter, using its own language.
    • interactive mode
      • You type commands on the command line and get the results in the work space
      • Use who to find what variables you have in the work space.
      • whos gives extra information: sizes, types, memory used
    • scripts vs functions
      • Create the habit of logging your work in script files. You can copy paste from a file on the command line in order to execute this piece of code. Also you can run a whole script by typing the name of the file on the command line.
      • By convention, both Matlab scripts and functions are saved in files with extension .m (m-files).
      • The command pwd prints the current working directory. Matlab searches for files first in the current directory. Then, in directories that are on the Matlab path. You can add a directory to the path by the function path. Note that Matlab is case sensitive. which shows the path to the file that is called.
    • toolboxes
      • You can organize a number of files into a "toolbox", just collect the files in one directory
      • Add a table of contents by including in the same directory an m-file with the name of the toolbox.
      • There is no distinction between functions written in m-files and Matlab's build in functions (that are hard codded into the system).

Getting help

  • A line beginning with % is a comment. A line beginning with %% is a section title. Anything between %{ and }% is a comment.
  • help — prints usage information for a Matlab function. The first lines of the m-file that are comments is the usage information printed by the help function. Create the habit of documenting your functions and scripts. type types the full code of an m-file.
  • lookfor — searches for functions by keywords,

Input and output

  • Use semi-columns ; at the end of a line to suppress output. Check the format command for different formats of the output.
  • Load/save data from/to files with load and save.
  • Export figures to different formats with print.

The matrix data type

IEEE arithmetic

  • eps — the smallest number in the double floating point arithmetic
  • inf — a number bigger that the biggest number represented in double floating point (over/under flow)
  • NaN — "Not a Number", results from undefined operations, e.g., 0 / 0, used also to represent missing data

Special matrices and vectors

  • zeros(n, m) / ones(n, m) is the \(n\times m\) matrix of all zeros / ones
  • eye(n) is the identity matrix \(I_{n}\), eye(m, n) returns the matrix \(\begin{bmatrix} I_{m} & 0_{m,n-m} \end{bmatrix}\) if \(m < n\) and \(\begin{bmatrix} I_{m} \\ 0_{m-n,n} \end{bmatrix}\) if \(n < m\)
  • 1:n is the row vector \(\begin{bmatrix} 1&\cdots&n \end{bmatrix}\), 1:s:n increments by s instead of by 1
  • [] is an empty matrix

Matrix indexing

  • defining a matrix and selecting elements (illustrated by an example)
    • a = [1 2 3; 4 5 6] defines the matrix \(A = \begin{bmatrix} 1&2&3\\ 4&5&6 \end{bmatrix}\)
    • a(1, 2) is the element \(A_{12}\)
    • a(1, :) is the first row, a(:, 1) is the first column, a(1:2, 1:2) is the submatrix \(A = \begin{bmatrix} 1&2 \\ 4&5 \end{bmatrix}\)
    • a(1, :) = [] deletes the first row
  • a(:) is the (column-wise) vectorized matrix, in the example \(\text{vec}(A) = \begin{bmatrix} 1& 4& 2& 5& 3& 6 \end{bmatrix}^{\top}\). If a is indexed with a single integer a(i) the index refers to the element in the vector \(\text{vec}(A)\).
  • More generally, a(I), where I is a matrix of integers between 1 and \(mn\), is the matrix with the same size as \(I\), which \((i,j)\)th element is \(A(I(i,j))\).

Other useful operations

  • You can concatenate matrices of compatible dimensions [A11 A12; A21 A22] (block matrices)
  • * and ^ are the matrix product and power. Element-wise matrix operations are preceded: a .* a.
  • Adding a scalar to a matrix adds the scalar to all elements.
  • Use reshape to reorganize the data in a matrix of different size with the same number of elements, e.g., for the example above reshape(a, 3, 2) gives the matrix \(\begin{bmatrix} 1&5\\ 4&3\\ 3&6 \end{bmatrix}\)
  • Use find to find elements in a matrix that satisfy a condition, e.g., [I, J] = find(a > 4)

Linear algebra

  • The generalization of the scalar division to matrices is used for solving systems of linear equations.
    1. square nonsingular matrix: x = a \ b is the unique solution \(x = A^{-1}b\) (computed in a better way than inverting \(A\) and multiplication of the inverse with \(b\))
    2. tall full column rank matrix: x = a \ b is the least-squares approximate solution \(x = (A^{\top}A)^{-1}A^{\top}b\), see also lscov
    3. far full row rank matrix: x = a \ b is a solution with \(n-m\) elements set to zero (not the least norm solution \(x = A^{\top} (AA^{\top})^{-1} b\))
  • eig and svd — eigenvalues and singular values decompositions, eigs and svds are used to compute selected eigenvalues and singular values and vectors (for large-scale problems)
  • rank, null, pinv — rank, null space and pseudo-inverse

Toolboxes

Optimization toolbox

  • functions can be defined in m-files or as implicit functions f = @(x) x ^ 2;
  • a wide rande of local optimization problems and methods, see the user manual or type help optim
  • numerical approximation of derivatives

Control toolbox

  • analysis and design of linear time-invariant systems, see the user manual or type help control
  • state-space, transfer function, zero-pole-gain objects
  • Kalman filter design, pole-placement, linear-quadratic optimal control

Identification toolbox

  • identification of linear time-invariant and certain classes of nonlinear systems, see the user manual or type help ident
  • parametric and nonparametric models / methods
  • subspace, instrumental variable, and prediction error methods
